# Звіт до практичної роботи №4

## Відповіді на контрольні запитання

**1. У чому полягає головна різниця між керованими та некерованими компонентами?**
Головна різниця полягає в тому, що керовані компоненти зберігають свій актуальний стан у React (через хук `useState`), і React є «єдиним джерелом істини» для значення поля. Кожна зміна вводу оновлює стан. Некеровані компоненти покладаються на внутрішній стан самого DOM-вузла, а дані з них зчитуються лише тоді, коли це необхідно (наприклад, при сабміті), зазвичай за допомогою `useRef`.

**2. Для чого в React-додатках використовується хук useRef?**
Хук `useRef` використовується для прямого доступу до DOM-елементів, коли потрібно виконати дії поза стандартним потоком даних React (наприклад, для примусового встановлення фокусу на інпут, роботи з медіа-плеєрами або сторонніми бібліотеками). Також він корисний для роботи з некерованими компонентами або для зберігання значень, які не повинні викликати перерендер компонента при зміні.

**3. Чому важливо використовувати event.preventDefault() при обробці сабміту форми?**
Виклик `event.preventDefault()` критично важливий, оскільки він блокує стандартну поведінку HTML-форми, яка полягає у відправці GET чи POST запиту на сервер і повному перезавантаженні сторінки. У Single Page Application (SPA), таких як React-додатки, ми хочемо обробити дані локально в JavaScript без перезавантаження, щоб зберегти поточний стан програми та забезпечити плавний досвід користувача.
