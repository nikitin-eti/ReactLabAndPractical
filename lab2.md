# Звіт до лабораторної роботи №2

## Код рендерингу списку в `App.jsx`

```jsx
<div style={{ maxWidth: '600px', width: '100%' }}>
  {postsData.map((post) => (
    <Post
      key={post.id}
      id={post.id}
      author={post.author}
      avatar={post.avatar}
      content={post.content}
      date={post.date}
      likes={post.likes}
    />
  ))}
</div>
```

## Детальне пояснення про атрибут `key`

Видалення атрибута `key` порушить алгоритм Reconciliation (узгодження) в React. Коли змінюється стан списку (додаються, видаляються або сортуються елементи), React використовує `key` для того, щоб точно знати, який саме компонент змінився. Якщо `key` відсутній або використовується нестабільний ключ (наприклад, індекс масиву), React не зможе ефективно відстежувати зміни. Це може призвести до:

- Зайвих перемалювань (re-renders), знижуючи продуктивність.
- Втрати внутрішнього стану компонентів (наприклад, якщо один із постів мав би стан відкритого коментаря, він міг би "перескочити" на інший пост).
  Саме тому React попереджає в консолі розробника: `Warning: Each child in a list should have a unique "key" prop`.

## Відповіді на контрольні запитання

**1. Навіщо потрібен проп key у списках React?**
Проп `key` потрібен для ідентифікації елементів та правильної роботи алгоритму узгодження (Reconciliation) при оновленні віртуального DOM. Він допомагає React визначити, які елементи були змінені, додані або видалені, що робить оновлення інтерфейсу ефективним і точним.

**2. Чому не рекомендується використовувати індекс масиву (index) як ключ?**
Це призводить до критичних помилок при зміні порядку елементів, їх додаванні або видаленні зі списку. Оскільки індекси змінюються при мутації масиву, React може помилково зіставити старий стан із новим елементом, що призведе до візуальних багів і втрати локального стану всередині компонентів списку.

**3. У чому різниця між імперативним створенням списку (цикл for) та декларативним (map)?**
Декларативний підхід (`map`) описує кінцевий результат (що ми хочемо отримати), не заглиблюючись у деталі ітерації, повертаючи новий масив елементів. Ітерація є чистою функцією. Натомість імперативний підхід (`for`) вимагає детального опису кожного кроку (як це зробити): створення початкового масиву, керування лічильником, додавання (push) кожного елемента по черзі.

**4. Як передати всі властивості об'єкта в компонент одним рядком?**
За допомогою синтаксису Spread attributes: `{...item}`. Це дозволяє розгорнути всі ключі та значення об'єкта і передати їх у вигляді окремих пропсів у компонент, роблячи код коротшим і читабельнішим.
