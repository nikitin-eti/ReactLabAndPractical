# Звіт до лабораторної роботи №1

## Код компонента `App.jsx`

```jsx
import React from 'react';
import './App.css';
import Header from './components/Header';
import Card from './components/molecules/Card/Card';
import Input from './components/atoms/Input/Input';
import Button from './components/atoms/Button/Button';

function App() {
  return (
    <>
      <Header />
      <main
        style={{
          padding: '50px 20px',
          display: 'flex',
          justifyContent: 'center',
        }}
      >
        <Card>
          <h2
            style={{
              color: '#22d3ee',
              fontFamily: 'monospace',
              marginBottom: '30px',
              textTransform: 'uppercase',
              textAlign: 'center',
              textShadow: '0 0 10px rgba(34, 211, 238, 0.5)',
            }}
          >
            Авторизація терміналу
          </h2>
          <Input type="email" label="Email" placeholder="user@system.io" />
          <Input type="password" label="Пароль" placeholder="••••••••" />
          <div
            style={{
              display: 'flex',
              gap: '15px',
              marginTop: '20px',
              width: '100%',
            }}
          >
            <Button variant="primary">Увійти</Button>
            <Button variant="secondary">Реєстрація</Button>
          </div>
        </Card>
      </main>
    </>
  );
}

export default App;
```

## Відповіді на контрольні запитання

### Що таке Atomic Design і навіщо ми розділяємо компоненти на атоми та молекули?

Atomic Design (Атомарний дизайн) — це методологія створення дизайн-систем і архітектури компонентів у веброзробці, яка поділяє інтерфейс на базові складові за аналогією з хімією: атоми, молекули, організми, шаблони (templates) та сторінки (pages).

Розділення на атоми та молекули робиться для:

- **Перевикористання коду**: дрібні компоненти (атоми, як-от кнопки чи інпути) можна використовувати в усьому застосунку не задумуючись про їх внутрішню реалізацію.
- **Підтримки консистентності**: єдиний стиль інтерфейсу гарантується на рівні базових будівельних блоків.
- **Масштабованості**: легше збирати складні інтерфейси (організми або сторінки) з уже готових, перевірених і стилізованих "цеглинок".

### Як працюють CSS Modules і як вони вирішують проблему глобальних імен класів?

CSS Modules — це підхід, за якого всі імена класів та анімації у CSS файлі автоматично компілюються (генеруються) з додаванням унікальних хешів (наприклад, `.button_a1b2c`). Під час імпорту такого файлу в компонент (через `import styles from './Button.module.css'`), розробник використовує об'єкт стилів (як-от `styles.button`), що дозволяє інструменту збирання проєкту (наприклад Vite/Webpack) підставити правильне унікальне ім'я класу у DOM.

Це вирішує проблему глобальної видимості: розробник може використовувати однакові прості імена класів (наприклад, `.wrapper`, `.title`) у різних файлах, і вони ніколи не конфліктуватимуть між собою, оскільки кінцевому класу буде присвоєно унікальне згенероване ім'я.

### Що таке props.children і в якому компоненті цієї лабораторної роботи ми його використали?

`props.children` — це спеціальна властивість в React, яка дозволяє передавати дочірні елементи або компоненти всередину іншого компонента. Це виступає в ролі "слота", куди React рендерить весь вміст, розміщений між відкриваючим і закриваючим тегами компонента.

У цій роботі ми використали `children` у:

- Компоненті `<Button>` для відображення тексту всередині кнопки (наприклад, `Увійти`, `Реєстрація`).
- Компоненті `<Card>` для рендерингу всього вмісту форми авторизації (заголовка, інпутів та кнопок), щоб зробити Card універсальним контейнером-обгорткою.

### Чому атрибут HTML class у JSX записується як className?

У React код записується за допомогою синтаксису JSX, який є розширенням JavaScript. Слово `class` є зарезервованим ключовим словом у JavaScript (використовується для створення класів, що було введено ще в стандарті ES6 / ES2015). Щоб уникнути синтаксичних конфліктів, розробники React вирішили використовувати `className` як аналог атрибута `class` в DOM API (`element.className`). Відповідно, React при рендерингу JSX у звичайний HTML автоматично перетворює `className` на HTML-атрибут `class`.
