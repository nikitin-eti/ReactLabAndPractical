# Звіт до практичної роботи №7: Параметри URL та Пошук

## Відповіді на контрольні запитання

**1. Яка різниця між параметрами шляху (URL Params) та параметрами запиту (Query Params)?**
URL-параметри шляху (Path Parameters, як `/users/42`) використовуються для ідентифікації конкретного ресурсу чи сторінки, тоді як параметри запиту (Query Parameters, як `/users?sort=asc`) застосовуються для модифікації відображення списку, наприклад, для фільтрації, сортування чи пагінації.

**2. Що робить хук useParams() та яка умова його правильної роботи?**
Хук `useParams()` витягує динамічні параметри з поточного URL; для його роботи необхідно обов'язково використати двокрапку перед назвою змінної у визначенні маршруту, наприклад `<Route path="/post/:postId">`.

**3. Чому зберігати стан пошуку в URL краще, ніж у useState()?**
Зберігання стану пошуку в URL через `useSearchParams` є кращим рішенням, оскільки це дозволяє користувачеві зберегти поточний вигляд сторінки в закладки або поділитися посиланням з іншими, при цьому всі фільтри та сортування автоматично відновляться при переході за лінком.

**4. Чи викликає фільтрація масиву через метод filter() проблеми з продуктивністю?**
Так, фільтрація через метод `filter()` прямо перед поверненням JSX відбувається при кожному рендері компонента. Для масивів на 10 000+ елементів це викличе проблеми з продуктивністю; щоб цього уникнути, слід використати хук `useMemo` для кешування відфільтрованого масиву.

**5. Чим об'єкт URLSearchParams у нативному JS відрізняється від хука useSearchParams у React Router?**
У базовому JavaScript параметри після знака питання зберігаються і обробляються через об'єкт `URLSearchParams`. React Router надає хук `useSearchParams`, який обгортає цей нативний API і повертає масив із двох елементів (поточні параметри та функцію для їх оновлення), працюючи за знайомим принципом `useState`.
