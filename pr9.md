# Звіт до практичної роботи №9: Оптимізація продуктивності

## Відповіді на контрольні запитання

**1. Яка різниця між хуком useMemo та функцією React.memo()?**
Хук `useMemo` використовується для кешування результатів важких обчислень або масивів/об'єктів всередині компонента, тоді як `React.memo()` - це HOC, який обгортає цілий компонент і кешує його рендер, запобігаючи перемальовуванню, якщо пропси не змінилися.

**2. Що означає жовтий або червоний колір коміту (блоку) у Flamegraph профілювальника?**
Жовтий або червоний колір коміту в Flamegraph означає, що рендер компонента зайняв багато часу (це потенційне "вузьке місце"), тоді як зелений або сірий означає швидкий рендер або те, що компонент взагалі не перемальовувався завдяки оптимізації.

**3. Чому використання React.memo() часто вимагає також використання useCallback у батьківському компоненті?**
При кожному рендері батьківського компонента всі функції всередині нього створюються наново з новими посиланнями в пам'яті. Через це `React.memo` бачить, що проп (функція) змінився (порушується Referential Equality), і все одно перемальовує дочірній елемент. `useCallback` фіксує посилання на функцію, вирішуючи цю проблему.

**4. Чи завжди варто використовувати useMemo для будь-яких обчислень чи створення масивів?**
Ні, хуки мають накладні витрати на виділення пам'яті та порівняння масиву залежностей при кожному рендері. Якщо використати `useMemo` для кешування простого рядка або математичної операції 2+2, перевірка залежностей хука займе більше часу і ресурсів, ніж саме обчислення.

**5. Як дізнатися причину рендеру конкретного компонента у React Profiler і навіщо потрібен масив залежностей у useCallback?**
Цю функцію ("Record why each component rendered while profiling") можна увімкнути в налаштуваннях React Profiler (іконка шестірні), після чого у правій панелі при кліку на компонент буде вказано причину рендеру (наприклад, "Hook 1 changed"). Якщо всередині `useCallback` використовується стан `count`, масив залежностей обов'язково має бути `[count]`, інакше функція запам'ятає старе (застаріле) значення стейту через механізм замикань (closures).
